Rewrite `app/streamlit_app.py` to support the full conversational experience:

1. **Dual session state** (CRITICAL — per research pitfall #6):
   - Keep `st.session_state.messages` as list of dicts `{"role": ..., "content": ...}` for Streamlit chat display
   - Add `st.session_state.chat_history` as list of `HumanMessage`/`AIMessage` objects for the LangChain retriever
   - Add `st.session_state.previous_query` to track the last question for topic shift detection
   - Add `st.session_state.previous_query_embedding` to cache the previous query's embedding (per research pitfall #4 — avoid redundant API calls)
   - Initialize all four in the session state init block

2. **One-time chain initialization**:
   - Build the conversational chain once and store in `st.session_state.rag_chain` (avoid rebuilding per question)
   - Build the embeddings model once and store in `st.session_state.embeddings`
   - Load vector store once and store in `st.session_state.vector_store`
   - Only rebuild if not already in session state

3. **Query handling flow** (replace the existing question-handling block):
   - When question is received, call `ask(chain, question, st.session_state.chat_history, embeddings, previous_query=st.session_state.previous_query)`
   - If `result["rewritten_query"]` is not None, display "I understood your question as: [rewritten query]" in a `st.info()` box before the answer (per user decision)
   - Display the answer text via `st.markdown(result["answer"])`
   - If `result["found"]` and `result["citations"]` exist, render expandable citations:
     - For each citation (max 2), create `st.expander(f"{citation['chapter']}, Page {citation['page']}", expanded=False)` containing `st.markdown(f"> {citation['passage']}")` showing the retrieved textbook passage
   - If not `result["found"]`, the answer already contains the fallback text from the prompt/chain
   - After displaying, update both session state lists:
     - Append to `messages`: user message dict + assistant message dict (answer text only, not citations — citations are rendered separately)
     - Append to `chat_history`: `HumanMessage(content=question)` + `AIMessage(content=result["answer"])`
     - Update `previous_query` to current question

4. **Sidebar updates**:
   - Update displayed config: TOP_K=6, CHUNK_SIZE=800, add SCORE_THRESHOLD=0.35
   - Add a "Clear Chat" button that resets all session state lists

5. **Off-topic detection**:
   - The system prompt handles off-topic detection — when the LLM receives no relevant context and the question is clearly outside science, it should use the OFF_TOPIC_RESPONSE pattern. Alternatively, add a simple keyword-based pre-check (Claude's discretion — keep it simple, the LLM prompt handles the heavy lifting).

6. **Fallback behavior** (per user decisions):
   - Binary confidence: The score threshold in the retriever (Plan 01) already handles this — if no chunks pass 0.35, the chain receives empty context, and the system prompt instructs the LLM to give the fallback response
   - The fallback suggests specific chapters or topics (handled in the updated SYSTEM_PROMPT)
   - Graceful partial answers on weak retrieval: When some chunks pass threshold but scores are low, the system prompt instructs to answer what it can and acknowledge gaps

7. **Chat replay**: When replaying chat history on page reload, replay from `st.session_state.messages` (display dicts). Do NOT replay citations — they are only shown on the turn they were generated (this keeps the replay clean and avoids storing large passage text in messages).
